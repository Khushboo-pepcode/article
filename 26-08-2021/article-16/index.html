UNDERSTANDING THE PROBLEM

Well, this problem states that we have to reverse an array. You may refer to the image below for understanding the problem.

Please note that we are not just printing the array in reverse, rather we are reversing it in the memory (as shown in the fig-1) so that the contents of the array are reversed inside it. You may refer to the question video to understand the problem if you face any difficulties.

PSEUDO CODE

Create an Auxiliary Array: Create an array of the same size as that of the input array.

Fill the Auxiliary Array: Fill this array by taking the variable at the end of the input array and traversing in reverse direction.

Fill back the original Array: Now copy all the elements back to the original array as it is from the Auxiliary array and the input array has been reversed.

But, friend, we are in the learning phase and it is always better to learn more, right? So, let’s discuss the very basic approach for this problem first. Have a look at the image given below:

So, does this image give you any ideas? Let’s see what we are doing here. We took an array b[ ] which was the same size as that of array a[ ]. Now, we are copying the values of a[ ] into b[ ] in reverse order.

We can do this simply, by taking a variable ‘i’ on the last index of a[ ] and a variable ‘j’ on the index 0 of array b[ ]. We do b [ j ]= a[ i ] and the value of a[ i ] gets copied to b[ j ] .

Then we increment the value of ‘j’ and decrement the value of ‘i’ and repeat the same procedure till ‘j’ completes the array in the forward direction and ‘i' completes the array in the backward direction.

As a result, we will get the reverse of a[ ] in b[ ] . But the problem states that we have to reverse the array a[ ]. This means that we can’t have the reverse of a[ ] just kept stored in b[ ]. Rather we have to reverse a[ ]. So we will copy back the values of b[ ] to a[ ] as shown in the figure. (fig-3).

So, as you can see from the figure, we have reversed the array a[ ] by copying back the elements from b[ ] to a[ ]. Dear reader, Please think that:

					            public static void nknights(int kpsf, int tk, 
					                                  boolean[][] chess, int lcno) {
					                if (kpsf == tk) {
					                    for (int row = 0; row < chess.length; row++) {
					                        for (int col = 0; col < chess.length; col++) {
					                            System.out.print(chess[row][col] ? "k\t" : "-\t");
					                        }
					                        System.out.println();
					                    }
					                    System.out.println();
					                    return;
					                }
					         
					                for (int i = lcno + 1; i < chess.length * chess.length; i++) {
					                    int row = i / chess.length;
					                    int col = i % chess.length;
					         
					                    if (chess[row][col] == false 
					                              && IsKnightSafe(chess, row, col)) {
					                        chess[row][col] = true;
					                        nknights(kpsf + 1, tk, chess, row * chess.length + col);
					                        chess[row][col] = false;
					                    }
					                }
					            }

Analysis
Time Complexity

In the recursion tree, we are having knights as levels, and the choices as selecting an empty box. Since levels are n (knights) and the total cells (choices) are n^2, hence the total time complexity should be O(n^2 * n^2 * …. n times) = O(n2n). But before placing any knights, we are checking whether placing it will result in valid configuration or not. Checking for validity requires checking 4 cells. Hence, the time complexity will be O( (4 * n^2) * (4 * n^2) * …. n times) = O((4n^2)n) = O(n2n).

Space Complexity

Since, the maximum depth of recursion is equal to the number of knights = n, hence the space complexity will be O(n), as recursion takes function call stack space.
